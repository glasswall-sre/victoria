victoria.config._print_validation_err(err: ValidationError, name: str, use_log: bool = True) → None¶

Internal function used for printing a validation error in the Schema.

Parameters

  • err (ValidationError) – The error to log.

  • name (str) – A human-readable identifier for the Schema data source. Like a filename.

  • use_log (bool) – Whether to use the logging package to log the error. This is needed because sometimes a validation error might occur when loading the logging config, before it’s configured.

victoria.config.load_plugin_config(plugin: Plugin, cfg: Config) → object¶

Load the config of a plugin from the main Victoria config.

Parameters

  • plugin (Plugin) – The plugin to load the config for.

  • cfg (Config) – The config file to load the config from.

Returns

The loaded config object. It will be the same type as whatever

the plugin’s config marshmallow schema will marshal it to, or None if there was some error loading the plugin config.

Return type

object

Raises

ValueError – if the plugin didn’t have a config schema.
